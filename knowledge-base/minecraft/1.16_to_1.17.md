## Version: 1.16.5 → 1.17.1

> **Java requirement:** Java 16 (1.17 dropped Java 8/11 support)
> **Source:** https://minecraft.wiki/w/Java_Edition_1.17

---

### Method Renames

- old: `KeyBinding`
- new: `KeyMapping`
- source: Mojang official mappings rename; Verified via compile (MC migration)

---

### Method Signature Changes

- before: `Screen.addButton(Button button)`
- after: `Screen.addRenderableWidget(Widget widget)`
- source: Fabric API / Minecraft source; Verified via compile (MC migration)

---

### API Changes

- description: Logging framework changed from Log4j direct imports to SLF4J
- impact: Replace `import org.apache.logging.log4j.Logger` and `LogManager.getLogger()` with `import org.slf4j.Logger` and `LoggerFactory.getLogger()`
- source: Minecraft 1.17 modernized logging; Verified via compile (MC migration)

- description: `KeyBinding` class renamed to `KeyMapping` (Mojang official mappings applied)
- impact: All keyboard input code must rename class references and import
- source: Mojang mappings; Verified via compile (MC migration)

---

### Common Migration Fixes

1. Find/replace `KeyBinding` → `KeyMapping` everywhere
2. Find/replace `addButton(` → `addRenderableWidget(`
3. Replace Log4j imports with SLF4J (see java/java17.md for details)
4. Bump `fabric-loader` to `0.11.7+build.2`, `fabric-api` to `0.46.1+1.17`
