## Version: 1.19.2 → 1.19.3

> **Java requirement:** Java 17 (unchanged)
> **Source:** https://minecraft.wiki/w/Java_Edition_1.19.3

---

### Widget API Changes

**Button.builder() fluent API added:**
- description: `Button` now uses a fluent builder API. Old direct constructor form no longer available.
- before: `new Button(x, y, w, h, label, onPress)`
- after: `Button.builder(label, onPress).bounds(x, y, w, h).build()`
- source: Fabric porting guide; Verified via compile (MC 1.19.3)

**AbstractWidget public field accessors added:**
- description: `getX()` and `getY()` getter methods were added. Pre-1.19.3 used public `x` and `y` fields.
- before (pre-1.19.3): `widget.x`, `widget.y`
- after (1.19.3+): `widget.getX()`, `widget.getY()`
- source: Fabric porting guide; Verified via compile (MC 1.19.3)

**EditBox.setHint() added:**
- description: `setHint(Component)` method added on `EditBox`. Not available pre-1.19.3.
- source: Fabric API changelog

**setFocused() visibility change:**
- description: `setFocused(boolean)` became protected in 1.19.3. Use `Screen.setFocused(GuiEventListener)` instead.
- before: `nameField.setFocused(true);`
- after: `this.setFocused(nameField);`
- source: Verified via compile (MC 1.19.3)

---

### Common Migration Fixes

1. Replace `new Button(x, y, w, h, label, onPress)` → `Button.builder(label, onPress).bounds(x, y, w, h).build()`
2. Replace `widget.x` / `widget.y` → `widget.getX()` / `widget.getY()`
3. Replace `nameField.setFocused(true)` → `this.setFocused(nameField)`
4. Remove or guard any `editBox.setHint(...)` calls if downgrading to pre-1.19.3
