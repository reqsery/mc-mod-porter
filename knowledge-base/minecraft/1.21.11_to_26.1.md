## Version: 1.21.11 → 26.1

> **Java requirement:** Java 25 (upgraded from Java 21)
> **Release date:** March 24, 2026
> **Source:** https://minecraft.wiki/w/Java_Edition_26.1
> **Fabric announcement:** https://fabricmc.net/2026/03/14/261.html
> **NeoForge announcement:** https://neoforged.net/news/26.1release/

---

### Version Format Change

Minecraft switched from `1.x.x` to a calendar-based `YY.drop.hotfix` format.
- `1.21.11` was the last 1.x release
- `26.1` is the first 26.x release (year 2026, first content drop)
- Hotfixes: `26.1.1`, `26.1.2` etc.

---

### Java-related Changes

- **Java 25 required** (was Java 21 for 1.21.x)
- Update all Gradle toolchain configs: `languageVersion = JavaLanguageVersion.of(25)`
- Source: https://minecraft.wiki/w/Java_Edition_26.1

---

### Code Obfuscation Removed

- description: 26.1 is the first fully unobfuscated Minecraft release
- impact: No Yarn or Mojang mappings remapping needed. Official parameter names are available directly.
- impact: ALL mods from 1.21.11 and before require at minimum recompilation — no binary compatibility
- source: https://fabricmc.net/2026/03/14/261.html, https://minecraft.wiki/w/Java_Edition_26.1

---

### Build Script Changes (Fabric)

- before: `id 'fabric-loom' version '1.x'` + `mappings loom.officialMojangMappings()`
- after: `id 'net.fabricmc.fabric-loom' version '1.15'` — mappings line removed entirely
- before: `modImplementation "net.fabricmc.fabric-api:fabric-api:..."`
- after: `implementation "net.fabricmc.fabric-api:fabric-api:..."` (standard `implementation`, not `modImplementation`)
- source: https://docs.fabricmc.net/develop/porting/

### NeoForge Version Format Change

NeoForge adopted four-component versioning to match Minecraft:
- Format: `26.1.0.<neoforge-build>[-beta]`
- Example: `26.1.0.10-beta` = MC 26.1 (hotfix 0), NeoForge build 10, still beta
- Source: https://neoforged.net/news/26.1release/

---

### API Changes (Fabric)

- description: `HudRenderCallback` removed
- impact: Replace with `HudElementRegistry`
- source: https://fabricmc.net/2026/03/14/261.html

- description: `fabric-convention-tags-v1` module removed
- impact: Remove from dependencies; use updated tag conventions
- source: https://fabricmc.net/2026/03/14/261.html

- description: `fabric-loot-api-v2` module removed
- impact: Migrate to current loot API
- source: https://fabricmc.net/2026/03/14/261.html

- description: `ItemGroupEvents` renamed
- impact: `ItemGroupEvents` → `CreativeModeTabEvents`
- source: https://fabricmc.net/2026/03/14/261.html

- description: `ColorProviderRegistry` simplified
- impact: `ColorProviderRegistry` → `BlockColorRegistry`
- source: https://fabricmc.net/2026/03/14/261.html

- description: `TradeOfferHelper` replaced
- impact: Villager trading is now data-driven via datapacks; `TradeOfferHelper` removed
- source: https://minecraft.wiki/w/Java_Edition_26.1

---

### API Changes (NeoForge)

- description: `ChunkPos` constructor removed
- impact: `new ChunkPos(blockPos)` → `ChunkPos.containing(blockPos)`
- source: https://neoforged.net/news/26.1release/

- description: `ItemStackTemplate` introduced for pre-registry item handling
- impact: Items constructed before registries load must use `ItemStackTemplate`
- source: https://neoforged.net/news/26.1release/

---

### Mapping Changes

- description: Yarn mappings no longer officially supported for 26.x
- impact: Fabric projects must use Mojang's official mappings (or no mappings, since code is unobfuscated)
- source: https://fabricmc.net/2026/03/14/261.html

---

### Data Changes

- description: `#dirt` block tag split
- impact: Tag `#dirt` split into `#dirt`, `#mud`, `#moss_blocks`, `#grass_blocks` — update any tag references
- source: https://minecraft.wiki/w/Java_Edition_26.1

---

### Common Migration Fixes

1. Update Java to 25 in Gradle toolchain
2. Change Loom plugin ID: `fabric-loom` → `net.fabricmc.fabric-loom`
3. Remove `mappings` line from build.gradle (no longer needed)
4. Change `modImplementation` → `implementation` for Fabric API
5. Replace `HudRenderCallback` → `HudElementRegistry`
6. Replace `ItemGroupEvents` → `CreativeModeTabEvents`
7. Replace `ColorProviderRegistry` → `BlockColorRegistry`
8. Remove deprecated modules: `fabric-convention-tags-v1`, `fabric-loot-api-v2`
9. Update NeoForge build tool to ModDevGradle 2.0.141+ or NeoGradle 7.1.21+
10. Recompile all mods — no binary compatibility with pre-26.1
