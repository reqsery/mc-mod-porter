## Version: 1.21.5 → 1.21.6

> **Java requirement:** Java 21 (unchanged)
> **Source:** https://minecraft.wiki/w/Java_Edition_1.21.6

---

### Rendering Changes

**GuiGraphics.blit — RenderType::guiTextured removed:**
- description: The `RenderType::guiTextured` method reference was removed. `blit` now uses fractional UV coordinates (0.0–1.0) without a `RenderType` argument.
- before (1.21.4–1.21.5): `graphics.blit(RenderType::guiTextured, skin, x, y, uPx, vPx, w, h, texW, texH)`
- after (1.21.6+): `graphics.blit(skin, x, y, w, h, u0, v0, u1, v1)` where u0/v0/u1/v1 are 0.0–1.0
- source: Verified via compile (MC 1.21.6)

**RenderType import:**
- Remove `import net.minecraft.client.renderer.RenderType;` if it was only used for `::guiTextured`
- source: Verified via compile (MC 1.21.6)

---

### Server API Changes

**ServerPlayer.server field made private:**
- description: The `server` field on `ServerPlayer` was made private in 1.21.6
- before: `player.server` or `sender.server`
- after: `((ServerLevel) player.level()).getServer()`
- source: Verified via compile (MC 1.21.6)

---

### Common Migration Fixes

1. Remove `RenderType::guiTextured` first arg from all `graphics.blit(...)` calls
2. Convert pixel UV coordinates to fractional (divide by texture size, e.g. `8f / 64f`)
3. Remove `import net.minecraft.client.renderer.RenderType;` if unused
4. Replace `player.server` → `((net.minecraft.server.level.ServerLevel) player.level()).getServer()`
5. Update `fabric-api` to `0.124.0+1.21.6`
