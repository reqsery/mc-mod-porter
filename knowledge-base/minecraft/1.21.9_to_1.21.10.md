## Version: 1.21.9 → 1.21.10

> **Java requirement:** Java 21 (unchanged)
> **Source:** https://minecraft.wiki/w/Java_Edition_1.21.10

---

### Input API Changes (Fabric)

**KeyMapping.Category (Fabric side):**
- description: `KeyMapping` constructor on Fabric now requires `KeyMapping.Category` instead of a `String` category
- note: NeoForge already changed this in 1.21.9. This is the Fabric side of the same change.
- before: `new KeyMapping("key.mymod.action", type, key, "key.categories.misc")`
- after: `new KeyMapping("key.mymod.action", type, key, KeyMapping.Category.register(ResourceLocation.fromNamespaceAndPath("mymod", "mymod")))`
- source: Verified via compile (MC 1.21.10 / Fabric)

---

### Entity World Access (Fabric 1.21.9/1.21.10)

- description: `Entity#getWorld` renamed to `Entity#getEntityWorld`
- impact: Affects almost all mods that access the world from an entity
- before: `entity.getWorld()`
- after: `entity.getEntityWorld()`
- source: https://fabricmc.net/2025/09/23/1219.html

---

### Client API Changes

- description: `MinecraftClient.IS_SYSTEM_MAC` renamed to `SystemKeycodes.IS_MAC_OS`
- before: `MinecraftClient.IS_SYSTEM_MAC`
- after: `SystemKeycodes.IS_MAC_OS`
- source: https://fabricmc.net/2025/09/23/1219.html

---

### Resource Loader API Rework (Fabric)

- description: `ResourceManagerHelper` registration API changed
- before: `ResourceManagerHelper.get(ResourceType.SERVER_DATA).registerReloadListener(new CustomReloader())`
- after: `ResourceLoader.get(ResourceType.SERVER_DATA).registerReloader(Identifier.of("modid", "name"), new CustomReloader())`
- source: https://fabricmc.net/2025/09/23/1219.html

---

### Block Entity Rendering (Fabric)

- description: `BlockEntityRenderer` signature completely changed — now uses `BlockEntityRenderState` and `OrderedRenderCommandQueue`
- note: Manual migration required — too mod-specific for automatic porting
- source: https://fabricmc.net/2025/09/23/1219.html

---

### Common Migration Fixes

1. Replace `entity.getWorld()` → `entity.getEntityWorld()`
2. Replace `MinecraftClient.IS_SYSTEM_MAC` → `SystemKeycodes.IS_MAC_OS`
3. (Fabric) Update `String` KeyMapping category → `KeyMapping.Category.register(ResourceLocation.fromNamespaceAndPath(...))`
4. Add `import net.minecraft.resources.ResourceLocation;` if not already present
5. Update `fabric-api` to `0.138.4+1.21.10`, `fabric-loader` to `0.18.4`, Loom to `1.11-SNAPSHOT`
